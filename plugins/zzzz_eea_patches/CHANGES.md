## Detailed Changes

### Change 1: Ticket Count

**Before:**
```erb
<% if tickets = HelpdeskTicket.includes(:issue => [:project]).where(:projects => {:id => @project}) %>
  ...
  <%= sprite_icon(..., :count => tickets.count) %>
```

**After:**
```erb
<% ticket_count = HelpdeskTicket.joins(:issue).where(:issues => { :project_id => @project.id }).count %>
...
<%= sprite_icon(..., :count => ticket_count) %>
```

**Why this helps:**
- **Before:** Loads 17,156 records with LEFT JOINs, then counts in Ruby
- **After:** Database executes COUNT() using index, returns single integer

### Change 2: Customer Count

```erb
<% customers = Contact.includes(:tickets => :project).where(:projects => {:id => @project}) %>

<%= sprite_icon(..., :count => customers.count) %>
```

**After:**

## SQL Comparison

### Original Query (Generated by Rails)

```sql
SELECT `helpdesk_tickets`.`id` AS t0_r0, 
       `helpdesk_tickets`.`contact_id` AS t0_r1,
       -- ... 19 columns from helpdesk_tickets
       `issues`.`id` AS t1_r0,
       `issues`.`project_id` AS t1_r1,
       -- ... 24 columns from issues
       `projects`.`id` AS t2_r0,
       `projects`.`name` AS t2_r1
       -- ... 18 columns from projects
FROM `helpdesk_tickets` 
LEFT OUTER JOIN `issues` ON `issues`.`id` = `helpdesk_tickets`.`issue_id` 
LEFT OUTER JOIN `projects` ON `projects`.`id` = `issues`.`project_id` 
WHERE `projects`.`id` = 161
```

**Result:** 17,156 rows × 61 columns = 1,046,516 data points  
**Time:** >120 seconds (timeout)

---

### Optimized Query (Generated by Rails)

```sql
-- Ticket count:
SELECT COUNT(*) 
FROM `helpdesk_tickets` 
INNER JOIN `issues` ON `issues`.`id` = `helpdesk_tickets`.`issue_id` 
WHERE `issues`.`project_id` = 161

-- Customer count:
SELECT COUNT(DISTINCT `helpdesk_tickets`.`contact_id`) 
FROM `helpdesk_tickets` 
INNER JOIN `issues` ON `issues`.`id` = `helpdesk_tickets`.`issue_id` 
WHERE `issues`.`project_id` = 161 
AND `helpdesk_tickets`.`contact_id` IS NOT NULL
```

**Result:** 2 integers  
**Time:** <100ms

---

## Index Usage

**Optimized queries use:**
1. `issues_project_id` index on `issues(project_id)` - for WHERE clause
2. `index_helpdesk_tickets_on_issue_id_and_contact_id` on `helpdesk_tickets(issue_id, contact_id)` - for JOIN and DISTINCT
